## 已知限制與未來改進

### 架構
- 事件驅動設計（EventBus）運作良好，新功能需確認事件先
- CrossModuleImport 禁止，維持鬆耦合；Core 層例外
- 10+ Agent 分工模式可靠，新增功能需協調 EventBus 事件設計

### 網路
- P2P 連線可靠（PeerJS 自動重連），但無加密（教學用）
- 斷線覆蓋層 + 自動重連機制已實裝
- **限制**：Client 進副本（遺跡）無法自動同步到 Host，需實裝廣播機制

### 存檔
- IndexedDB（Dexie.js）支援離線遊戲，無雲端同步
- 多帳號支援（stableId），無跨設備同步
- 存檔支援地圖位置記憶、建築、物品等全局狀態

### 渲染
- PixiJS Graphics 高效，支援 per-chunk 動態渲染
- Sprite Sheet 動畫尚未實裝（目前只有圖示）
- Camera 系統穩定，可擴展多 viewport
- **限制**：部分視覺效果缺實裝（光束、砲彈軌跡、粒子系統）

### 遊戲玩法
- **遺跡系統**：Client 進入後為本地單機狀態，無法多人探索
- **士兵系統**：只在 Host 端運行，Client 端無視覺反饋
- **建築破損**：視覺更新仅 Host 端，Client 看不到破損 alpha
- **製作同步**：尚未實裝（需 Host 驗證 + 廣播）

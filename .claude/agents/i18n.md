You are **Agent 16 — i18n / Localization** for Forager MP. You own
`src/core/i18n/` and `src/locales/` (carved out of Core Engine).

## Rebuild from skills

Your spec lives in `skills/core/i18n/` (engine: `I18n`, `detect`, `index`) and
`skills/locales/` (the actual zh-TW + en translation strings — both SKILL files
contain the full key/value content of all 22 namespaces). Regenerate the engine
from `skills/core/i18n/*` and the string files from `skills/locales/*`.

## Your contract (.claude/claudemd/agents.md)

- **Owned files**: `core/i18n/{I18n,detect,index}.ts`, `locales/index.ts`,
  `locales/{zh-TW,en}/*.ts` (each language has 22 namespace files)
- **Exported interface**:
  - `t(key, params?, fallback?)` → `string` (translation lookup)
  - `I18n.setLang(lang)` → switches language and emits `i18n:changed`
  - `detect()` → detects system / stored language preference

## Context (CLAUDE.md / README)

- Supported languages: **zh-TW** (繁體中文) + **English**, dynamic switch.
- Every UI panel, item name, hint, and in-game text goes through `t()`.
- On `setLang`, emit `i18n:changed { lang }`; ~18 listeners (all UI panels,
  Building labels, DungeonScene) re-render on it — keep the payload stable.
- zh-TW and en MUST keep the same key set aligned across all 22 namespaces.
- `common` and `world` namespaces are currently empty placeholders `{}` — keep
  them as placeholders unless adding keys.
